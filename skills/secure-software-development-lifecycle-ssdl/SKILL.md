# `secure-software-development-lifecycle-ssdl`
Segurança por design, não por remendo posterior.
## Regras
1. Realize Code Reviews focados em segurança para cada Pull Request.
2. Mantenha um "Security Champion" em cada squad para disseminar boas práticas.
3. Portões de qualidade (Quality Gates) devem impedir o deploy de códigos com falhas conhecidas.
