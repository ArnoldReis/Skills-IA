# `oas-openapi-specification-pro`
A linguagem padrão para descrever o ecossistema REST.
## Regras
1. Utilize as tags `paths`, `components` e `securitySchemes` para organizar de forma modular seu YAML/JSON.
2. Defina `example` e `description` ricos em todos os campos para reduzir o tempo de suporte aos desenvolvedores externos.
3. Utilize referências externas ($ref) para reaproveitar modelos de dados comuns entre diferentes endpoints.
