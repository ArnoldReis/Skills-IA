# `algorithm-complexity-analysis-big-o`
Medindo a performance conforme a entrada de dados cresce.
## Regras
1. Foque em algoritmos O(1) ou O(log n) para operações críticas de busca em grandes volumes de dados.
2. Evite a todo custo loops aninhados que resultem em O(n²) em caminhos de código acessados por usuários.
3. Escolha a estrutura de dados certa (Hash Table vs Array vs Linked List) baseada no perfil de acesso (leitura vs escrita).
