# `database-seeding-faker-tool`
Cenários de teste com dados que parecem reais.
## Regras
1. Utilize seeds fixas para garantir que os testes sejam reproduzíveis.
2. Gere nomes, emails e endereços que sigam padrões locais desejados.
3. Não utilize dados reais de clientes em ambientes de desenvolvimento.
