# `incident-retrospective-sharing`
Aprendendo com os erros coletivos para construir sistemas melhores.
## Regras
1. Publique os resultados dos Post-mortems em um repositório interno acessível a toda a equipe de engenharia.
2. Realize reuniões periódicas (ex: Monthly Reliability Review) para discutir as causas raízes mais comuns do mês.
3. Utilize os aprendizados para atualizar padrões de arquitetura e guias de codificação internos.
