# `secure-coding-patterns-guide`
Convertendo princípios de segurança em hábitos de codificação.
## Regras
1. Trate toda entrada de usuário como hostil; valide e neutralize antes de processar.
2. Use APIs parametrizadas para consultas de banco e comandos de sistema para evitar injeções.
3. Não confie em controles de segurança aplicados apenas no lado do cliente (Frontend); valide tudo no Backend.
