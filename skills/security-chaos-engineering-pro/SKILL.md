# `security-chaos-engineering-pro`
Fortalecimento através do caos controlado.
## Regras
1. Simule a queda de um serviço de autenticação chave para ver como o sistema se comporta.
2. Injet de lag ou falhas em APIs de segurança para testar fallbacks e logs.
3. Realize "Game Days" onde um cenário de invasão é simulado para treinar o time de resposta.
