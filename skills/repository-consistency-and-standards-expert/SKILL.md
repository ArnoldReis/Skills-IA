# `repository-consistency-and-standards-expert`
A marca da excelência através de padrões uniformes.
## Regras
1. Toda skill deve possuir um `SKILL.md` com regras de domínio e um `index.ts` com boilerplate funcional.
2. Utilize comentários "Why" (Porquê) no código para explicar decisões de arquitetura não óbvias.
3. Garanta que toda documentação esteja em Português-BR para manter a coesão linguística do projeto.
