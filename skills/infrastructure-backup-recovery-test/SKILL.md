# `infrastructure-backup-recovery-test`
Confiança absoluta na recuperação de desastres.
## Regras
1. Um backup não testado é um backup que provavelmente falhará.
2. Restaure backups em um ambiente isolado (Sanitized) toda semana.
3. Compare a integridade dos dados restaurados com a fonte original.
