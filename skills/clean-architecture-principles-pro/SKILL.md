# `clean-architecture-principles-pro`
Sistemas organizados por intenção e desacoplados do detalhe.
## Regras
1. Siga a Regra de Dependência: o código de dentro nunca deve saber nada sobre o código de fora.
2. Pense em Entidades, Casos de Uso (Interactors) e Apresentadores (Presenters) como camadas distintas.
3. Teste a lógica de negócios sem a necessidade de rodar o servidor web ou conectar ao banco.
