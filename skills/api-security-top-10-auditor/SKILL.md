# `api-security-top-10-auditor`
Proteção especializada para a principal porta de entrada do software moderno.
## Regras
1. Verifique o BOLA (Broken Object Level Authorization) garantindo que um usuário não acesse recursos de outros via ID.
2. Identifique o BFLA (Broken Function Level Authorization) conferindo se funções administrativas estão protegidas.
3. Evite a Exposição Excessiva de Dados (Excessive Data Exposure) retornando apenas o necessário no JSON.
