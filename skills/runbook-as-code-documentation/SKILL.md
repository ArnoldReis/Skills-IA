# `runbook-as-code-documentation`
Documentação operacional que executa e resolve problemas.
## Regras
1. Mantenha os runbooks no mesmo repositório do código para garantir que a documentação acompanhe a evolução do sistema.
2. Utilize ferramentas de automação (Ansible, Terraform, Scripts) que possam ser executadas com um único comando.
3. Teste os runbooks periodicamente para garantir que eles ainda são válidos após mudanças na infraestrutura.
